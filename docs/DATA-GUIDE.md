# Data Guide — Researcher Reference

This is the technical companion to the [README](../README.md). For someone who wants to use the data files in their own work, or verify the values used in this tool.

---

## Overview

This repository includes digitized, source-attributed datasets for MH370 satellite analysis. All BTO/BFO values and model constants have been cross-checked against primary sources. A verification log tracks confirmation status.

Two categories of data:

1. **Reference datasets** (`knowledge/`) — Verified input data: handshake measurements, model constants, satellite state vectors. These are the "ground truth" inputs to the analysis.
2. **Snapshot data** (`public/data/`) — Solver output and derived layers. These are generated by the Rust backend and exported as GeoJSON/JSON for the web viewer.

## Reference Dataset Files

Located in `knowledge/`:

| File | Contents | Format |
|---|---|---|
| `mh370_satcom_data.csv` | All handshake BTO/BFO values with source attribution per row | CSV with comment header |
| `mh370_model_constants.csv` | Every BFO/BTO model parameter, verified against primary sources | CSV with comment header |
| `mh370_reference_data.xlsx` | Comprehensive reference: sheets covering handshake data, model constants, per-arc corrections, satellite state vectors, searched areas, BFO equation reference, and verification log | XLSX |

### mh370_satcom_data.csv

Each row is a satellite communication event (handshake, log-on, phone call, ACARS).

| Column | Description |
|---|---|
| `arc` | Arc number (1-7) or `-` for non-arc events |
| `event` | Event type: `ges_handshake`, `aes_logon`, `phone_call_1`, `gate_logon`, etc. |
| `time_utc` | Timestamp in UTC (HH:MM:SS). Arcs 6-7 are on March 8 after midnight. |
| `bto_raw_us` | Raw burst timing offset in microseconds, as recorded |
| `bto_corrected_us` | BTO after applying the -4600 us log-on correction where applicable |
| `bfo_hz` | Burst frequency offset in Hz |
| `channel` | R1200 or C-channel |
| `bto_usable` | Whether BTO is usable for arc calculation (Y/N/Y_calibration) |
| `bfo_usable` | Whether BFO is usable for scoring (Y/N/Y_logon_request_only/Y_with_warmup_caveat) |
| `bto_sigma_us` | BTO noise standard deviation: 29 us for R1200, 62 us for R600 |
| `dfsat_afc_hz` | Satellite oscillator drift + AFC correction term per arc |
| `source` | Primary source(s) for this row's values |

**Primary sources:** Inmarsat SU logs (CNN release May 2014), Ashton et al. 2014 *J. Navigation* Table 6, Holland 2017 arXiv:1702.02432.

**Key notes:**
- Arcs 1 and 7 BTO values have -4600 us correction applied (Ashton 2014 Table 6 note 1) because they are AES-initiated log-on events.
- The 16:00 gate log-on BTO is used for calibration only (known position at KLIA).
- Arc 7 BFO of 182 Hz is from the first log-on request only. The second value (-2 Hz, 8 seconds later) has a warmup caveat.

### mh370_model_constants.csv

Each row is a model parameter used in BTO/BFO calculations.

| Column | Description |
|---|---|
| `parameter` | Parameter name (snake_case) |
| `value` | Numeric value |
| `unit` | Physical unit |
| `source` | Primary source document and location (table, page, section) |
| `notes` | Context, alternative values, caveats |
| `verified` | Whether cross-checked against the primary source (yes/no) |

### mh370_reference_data.xlsx

Multi-sheet workbook for comprehensive cross-referencing. Sheets include:
- Handshake data (mirrors the CSV with additional context)
- Model constants (mirrors the CSV)
- Per-arc satellite corrections
- Satellite state vectors
- Searched area boundaries
- BFO equation component reference
- Verification log (what has been confirmed, what remains open)

## Sonar Coverage Files

Located in `public/data/sonar_coverage/`:

| File | Contents | Features |
|---|---|---|
| `auv_coverage.geojson` | AUV Side Scan Sonar coverage footprint (Phase 2, 5m resolution) | 43 polygons |
| `deeptow_coverage.geojson` | Deep Tow Side Scan Sonar coverage footprint (Phase 2, 5m resolution) | 2 polygons |
| `gophoenix_coverage.geojson` | Go Phoenix / Dong Hai Jiu 101 Synthetic Aperture Sonar coverage (Phase 2, 5m resolution) | 12 polygons |
| `bathymetry_coverage.geojson` | Bathymetry survey coverage (Phase 1, 150m resolution) | 36 polygons |
| `all_coverage.geojson` | Union of all four layers into a single MultiPolygon | 1 feature |

**Derivation method:** These are derived from Geoscience Australia WMS raster data by requesting GetMap PNGs with transparency, extracting the alpha channel to create a binary mask, applying morphological closing to merge adjacent scan swaths, tracing contours, and simplifying to ~0.02 degree polygons. They are approximations of the actual scan coverage, not authoritative boundaries.

**Regeneration:** `python3 scripts/extract_sonar_coverage.py` (requires requests, Pillow, numpy, scipy, shapely, scikit-image, geojson).

**Attribution:** Governments of Australia, Malaysia and the People's Republic of China, 2018. CC BY 4.0.

See `public/data/sonar_coverage/README.md` for full technical details including WMS layer names and extraction parameters.

## Snapshot Data Files

Located in `public/data/`. These are generated by the Rust backend solver and exported for the static web viewer.

| File | Contents | Generated by |
|---|---|---|
| `arc_rings.geojson` | BTO distance rings from Inmarsat-3F1 at each handshake timestamp, with satellite drift correction | `export_snapshot` binary / `arcs.rs` |
| `candidate_paths.geojson` | Sampled flight paths through arcs 2-7, scored by BTO/BFO/fuel feasibility, coloured by path family | `export_snapshot` binary / `paths.rs` |
| `probability_heatmap.geojson` | Path density distribution along the 7th arc (beam search scoring output) | `export_snapshot` binary / `probability.rs` |
| `debris_points.geojson` | 43 confirmed and suspected debris find locations with metadata | Manually compiled from ATSB reports |
| `debris_drift.geojson` | Reverse drift corridors from debris find locations | `drift.rs` / `drift_transport.rs` |
| `debris_inversion_result.json` | Joint debris origin inversion — most likely splash point given all debris finds | `debris_inversion.rs` |
| `handshakes.json` | Satellite handshake timestamps and measurements for the BFO panel | `data.rs` |
| `anomalies.geojson` | Evidence markers: hydroacoustic events, satellite imagery, barnacle analysis | Manually compiled |
| `anomalies.json` | Anomaly metadata (descriptions, sources, assessment) | Manually compiled |
| `data_holidays.geojson` | Areas within the searched zone with missing or poor-quality sonar data | ATSB 2022 Data Review Report |
| `airspaces.geojson` | Flight Information Region boundaries, frozen to March 8 2014 | ICAO, manually encoded |

To regenerate the solver output files:

```bash
cd src-tauri && cargo run --bin export_snapshot
```

## Key Constants and Their Sources

| Constant | Value | Source | Notes |
|---|---|---|---|
| BFO bias | 150.0 Hz | Holland 2017 | SDU oscillator offset from 20 prior 9M-MRO flights. Ashton 2014 used 152.5 Hz; difference is within noise. |
| BFO noise sigma | 4.3 Hz | DSTG Table 5.1 | In-flight, outliers excluded. DSTG used 7 Hz for the accident flight model. |
| BTO noise sigma | 29 us (R1200) | DSTG Table 10.1 | R600 messages: 62 us. |
| BTO bias | -495,679 us | ATSB | Calibrated from pre-departure BTO at KLIA gate. |
| Log-on BTO correction | -4,600 us | Ashton 2014 Table 6 note (1) | Applied to AES-initiated log-on messages at 18:25 and 00:19. |
| AES compensation altitude | 36,210.12 km | DSTG Ch.5 p.29 | = 35,788.12 + 422 km. SDU design choice — it compensates range as if the satellite were at this altitude. |
| Nominal geostationary altitude | 35,788.12 km | DSTG Ch.5 p.29 | Standard geostationary altitude above surface. |
| Satellite nominal longitude | 64.5 deg E | DSTG Book | Inmarsat-3F1. Actual position drifts; ephemeris needed per arc. |
| Uplink frequency | 1,646.6525 MHz | Holland 2017 | L-band aircraft-to-satellite. |
| Perth GES location | 31.802 deg S, 115.889 deg E | DSTG Table 2.1 | Perth Ground Earth Station. |
| Speed of light | 299,792.458 km/s | NIST | Used for BTO range and Doppler calculations. |

## Known Gaps

- **DSTG Book Table 2.3 does not exist** — earlier documentation referenced it incorrectly. Numeric BTO/BFO values come from the Inmarsat SU logs, Ashton et al. 2014 Table 6, and Holland 2017 Table III.
- **DSTG 20-flight validation dataset** is not publicly available. The 4.3 Hz BFO sigma is adopted on trust from their published results (Table 5.1).
- **Ocean Infinity 2018 and 2025-2026 search data** is not available through Geoscience Australia. Coverage boundaries for those searches are approximate.
- **True FMS-reported ACARS position at 17:07** was never publicly released. The 5.27 deg N, 102.79 deg E used here is from the ACARS report, not the FMS fix.
- **HYCOM high-resolution currents** — drift currently uses OSCAR at 1/3 degree with ERA5 monthly wind. HYCOM support is stubbed but not yet active.
